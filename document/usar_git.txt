Configurando as credenciais do Git
Com o Git instalado, precisamos configurar as credenciais, que são as configurações de nome de usuário e e-mail. Isso é necessário para registrar as modificações no código, associando-as ao nosso e-mail e nome. Assim, ao verificar uma modificação, aparecerá que foi feita por determinado e-mail e usuário. Para um uso mais profissional, utilizamos as mesmas informações do GitHub. Antes de configurar as credenciais, criamos uma conta no GitHub, pois usaremos as informações fornecidas lá.

Para acessar o GitHub, voltamos ao Chrome e acessamos www.github.com. Na página inicial, podemos entrar ou criar uma conta. Recomendamos usar um e-mail ativo e profissional, pois muitas pessoas, incluindo recrutadores, podem visualizar nosso perfil no GitHub. Após criar a conta, usamos os mesmos dados de nome de usuário e e-mail para configurar as credenciais do Git.

Configurando o nome de usuário e e-mail

Adicionando e comitando arquivos
Durante a inicialização, o Git cria uma pasta oculta no sistema operacional para armazenar as modificações. O primeiro passo após a instalação é usar o comando:

git status
Copiar código
Esse comando informa o status dos arquivos na pasta. Ele indica se os arquivos já foram mapeados ou se precisam ser registrados na pasta oculta do Git. Para adicionar os arquivos, utilizamos o comando:

git add .
Copiar código
Ao escrever o comando git add, podemos passar arquivo por arquivo para adicionar e registrar. Por exemplo, podemos passar index.html e pressionar "Enter". Ao executar novamente o comando git status, o arquivo index.html será identificado, mas os outros ainda não. A melhor forma de adicionar todos os arquivos é usar o símbolo de ponto. Assim, ao escrever git add ., estamos informando ao Git que queremos adicionar todos os arquivos do diretório atual. Após pressionar "Enter" e executar git status novamente, o Git reconhecerá todos os arquivos.

Realizando o primeiro commit
Existe uma nomenclatura para isso: arquivos unstaged (não rastreados) e arquivos em stage (conhecidos, mas sem modificações registradas). Para registrar uma primeira versão do código, selecionamos os arquivos desejados e realizamos um commit, que é o registro de uma modificação no projeto. Selecionamos os arquivos para o commit e, em seguida, digitamos o comando:

git commit -m "Commit Inicial"
Copiar código
A escolha do idioma para as mensagens de commit pode ser pessoal ou acordada pela equipe de tecnologia.

Visualizando o histórico de commits
Após escrever a mensagem e pressionar "Enter", o Git mostrará que todos os arquivos não rastreados foram registrados. Podemos ver o log dessas modificações com o comando:

Esse comando exibirá informações como o autor do commit, e-mail e data. Para sair dessa área de informação, pressionamos a letra Q.

Utilizando o VSCode para gerenciar o Git
Se o terminal parecer confuso para executar comandos do Git, o VSCode oferece uma interface amigável. No menu lateral, o ícone de controle de código fonte conecta-se ao Git. Ele exibe alterações e listas de modificações. Podemos ver o que foi alterado clicando no documento, que mostrará as diferenças. Para preparar um arquivo para commit, clicamos no ícone de mais, movendo-o para a seção de alterações preparadas. Podemos escrever a mensagem diretamente no VSCode e confirmar, executando os comandos por trás dos panos.

Na linha do tempo, o VSCode mostrará os commits realizados. Podemos clicar em cada um para ver os arquivos modificados. O projeto está agora preparado para registrar modificações, o que é útil em caso de mudanças de escopo, permitindo navegar entre as alterações conforme necessário.
